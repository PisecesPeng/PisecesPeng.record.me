## 装饰器模式示例

> 1. 装饰器类和原始类继承同样的父类, 这样我们可以对原始类"嵌套"多个装饰器类.
> 2. 装饰器类是对功能的增强.

> 列出装饰器模式写法(非仅此写法):

```java
// 代理模式的代码结构(下面的接口也可以替换成抽象类)
public interface IA {
  void func();
}
public class A impelements IA {
  public void func() { 
    // do something...
  }
}
public class AProxy implements IA {
  
  private IA a;

  public AProxy(IA a) {
    this.a = a;
  }
  
  public void func() {
    // 新添加的代理逻辑
    a.func();
    // 新添加的代理逻辑
  }
}

// 装饰器模式与代理模式代码结构相似, 主要区别在于:
// 装饰器侧重点在于'增强调用功能',
// 代理侧重点在于'引入与调用功能不相关逻辑'.

// 装饰器模式的代码结构(下面的接口也可以替换成抽象类)
public interface IA {
  void func();
}
public class A implements IA {
  public void func() { 
    // do something...
  }
}
public class ADecorator implements IA {

  private IA a;

  public ADecorator(IA a) {
    this.a = a;
  }
  
  public void func() {
    // 功能增强代码
    a.func();
    // 功能增强代码
  }
}
```
